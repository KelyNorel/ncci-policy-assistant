# 2026 Medicare NCCI Policy Assistant

An AI-powered application for exploring the 2026 CMS Medicare National Correct Coding Initiative (NCCI) Coding Policy Manual. Built as a take-home assignment for Claritev's Agent Foundation Team / Innovation group.

---

## Features

### 💬 Chat
RAG-grounded Q&A over the full NCCI manual. Ask any coding policy question and get a cited, grounded answer with page references. The system admits when information is not found rather than hallucinating.

### 🔍 Policy Insights
Select from 14 predefined policy topics (or enter a custom topic) to generate a structured summary: Overview, Key Rules, Exceptions, and Relevant CPT Code Ranges — all grounded in the manual with relevance scores.

### 🔎 CPT Analyzer
- **Single code:** Enter one CPT code to understand applicable NCCI policies, global period, bundling considerations, and modifier guidance
- **Code pair:** Enter two CPT codes to analyze whether they can be reported together, with compatibility assessment and modifier options

### ⚠️ Conflict Detector
Enter up to 4 CPT codes billed together. The system analyzes all code pairs in parallel and produces a conflict summary grid with 🔴/🟡/🟢 indicators, detailed analysis per pair, and a downloadable report.

## Architecture

```
PDF (287 pages)
     │
     ▼
ingest.py       → pdfplumber extraction → section detection → chunking
                → cosine similarity deduplication (threshold=0.90)
                → all-MiniLM-L6-v2 embeddings → ChromaDB (local)
     │
     ▼
retriever.py    → query embedding → fetch 15 candidates
                → MMR re-ranking (diversity=0.4)
                → per-page limit (max 2 chunks/page)
                → minimum score filter (MIN_SCORE=0.42)
     │
     ▼
llm.py          → format context with [SOURCE N] labels
                → Claude Sonnet 4.6 API call
                → grounded response with inline citations
     │
     ▼
app.py          → Streamlit UI (4 tabs)
```

## Setup

### 1. Clone and create virtual environment
```bash
git clone <repo>
cd Claritev/Code
python3 -m venv venv
source venv/bin/activate
```

### 2. Install dependencies
```bash
pip install -r requirements.txt
```

### 3. Set up environment variables
Create a `.env` file in `Code/`:
```
ANTHROPIC_API_KEY=sk-ant-...
```

### 4. Add the PDF
Place the 2026 NCCI manual PDF in `Data/`:
```
Claritev/Data/2026_ncci_medicare_policy_manual_all-chapters.pdf
```

The PDF is available at:  
https://www.cms.gov/files/document/2026-ncci-medicare-policy-manual-all-chapters.pdf

### 5. Build the vector index
```bash
python ingest.py --pdf ../Data/2026_ncci_medicare_policy_manual_all-chapters.pdf
```

This takes ~5 minutes on CPU. The index is saved to `chroma_db/` and persists across sessions. To rebuild from scratch:
```bash
python ingest.py --pdf ../Data/2026_ncci_medicare_policy_manual_all-chapters.pdf --reset
```

### 6. Run the app
```bash
streamlit run app.py --server.fileWatcherType none
```

Open http://localhost:8501 in your browser.

## Technical Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Vector store | ChromaDB (local) | Zero infrastructure, persistent, no API keys |
| Embeddings | all-MiniLM-L6-v2 | Fast, local, no cost, good semantic similarity |
| LLM | Claude Sonnet 4.6 | Strong instruction-following, reliable citation behavior |
| Retrieval | MMR + page limit + score filter | Balances relevance and diversity, prevents redundant sources |
| Deduplication | Cosine similarity (threshold=0.90) | Handles verbatim boilerplate repeated across chapters |

---

## Evaluation

The system was evaluated against a 10-query gold standard set using Claude Sonnet 4.6 with the full PDF as ground truth (see `eval_set.md`).

**Result: 8 Pass / 1 Partial Pass / 1 Fail = 85% quality score**

Key findings:
- Strong hallucination resistance — system never fabricated information
- Correctly declined out-of-scope questions without citations
- One retrieval gap identified (Chapter XIII telehealth content)
- One partial pass due to chunk boundary issue at section start

---

## Known Limitations

- The NCCI manual repeats boilerplate text across chapters. Deduplication (cosine similarity > 0.90) removes 72 near-duplicate chunks but some redundancy may remain.
- Chunk boundaries occasionally split section content, causing the retriever to miss section-opening text.
- Chapter XIII coverage is incomplete in the current index.
- The system does not have access to the actual NCCI PTP edit tables (separate CMS files) — CPT Analyzer and Conflict Detector provide policy-based analysis, not lookup against the actual edit pairs.

## Development Approach

This application was built using an AI-native development workflow. Claude (claude.ai) served as the primary coding assistant throughout the build — generating initial code for all modules, diagnosing runtime errors, and implementing algorithms (MMR, cosine similarity deduplication, parallel execution).

All architectural decisions, quality judgments, parameter tuning, and output validation were performed by the developer. See `build_journal.md` for a detailed account of where AI helped, where it made mistakes, and how outputs were validated.
